# SETUP_GITHUB.md — 깃허브 연결 & 2-에이전트 협업 활성화 (경빈님 액션 가이드)

## 왜 하는가 (한 줄)

지금 저장소는 임시 클라우드 샌드박스에만 있어서 **회수되면 사라질 수 있다.** 깃허브에
올리면 ① 백업 ② 클로드 코드와의 공유 지점 ③ 컴퓨터 꺼져도 도는 자율 모드가 한 번에 해결된다.

---

## 당신이 할 일은 딱 하나: 깃허브 저장소 만들기 + 코워크가 push하게 해주기

방식은 둘 중 편한 쪽을 고르세요.

### 방식 1) 코워크가 직접 push (권장, 가장 매끄러움)

1. 깃허브에서 **빈 저장소**를 하나 만든다(예: `agentrelay`, private 권장).
2. 그 저장소에만 권한이 있는 **fine-grained Personal Access Token**을 발급한다
   (권한: Contents = Read and write, 그 저장소 하나로 스코프 제한).
3. 그 토큰과 저장소 URL을 이 대화에 붙여주면, 코워크가 샌드박스에서 원격을 설정하고
   현재까지의 전체 커밋을 push한다.
   - 보안: 토큰은 저장소 하나에만 쓰이고, 세션이 끝나면 샌드박스와 함께 폐기된다.
     불안하면 push 후 토큰을 즉시 폐기(revoke)해도 되고, 그 경우 이후 동기화는 방식 2로 전환.

### 방식 2) 당신이 직접 push (토큰을 주기 싫을 때)

1. 코워크가 방금 보낸 저장소 스냅샷(`agentrelay-kickoff-snapshot.tar.gz`)이나, 요청 시
   만들어줄 `git bundle`을 받는다.
2. 당신 컴퓨터에서 압축을 풀고:
   ```bash
   cd agentrelay
   git init && git add -A && git commit -m "import agentrelay"   # bundle이면 git clone
   git remote add origin <당신의-깃허브-저장소-URL>
   git push -u origin main
   ```
3. 이후 코워크는 git 대신 "산출물(문서/리뷰/제안)"을 이 대화로 전달하고, 코드 반영은
   당신과 클로드 코드가 담당(코워크는 읽기·조언 중심).

---

## 그다음: 클로드 코드를 주력 빌더로 붙이기

깃허브 저장소가 생기면, 아래 중 하나로 클로드 코드를 자율 개발자로 연결한다.

- **A. 클로드 코드 Routines (권장, 컴퓨터 꺼져도 됨)**: 클로드 코드에서 `/schedule`(또는
  데스크톱/웹)로 이 저장소를 대상으로 한 반복 작업을 만든다. 프롬프트는 "이 저장소를 pull하고
  `CLAUDE.md`/`BACKLOG.md`를 읽어 다음 항목을 구현해 PR을 열어라". Anthropic 인프라에서
  돌아 컴퓨터가 꺼져 있어도 진행된다.
- **B. 클로드 코드 GitHub App / Actions**: 저장소에 클로드 코드 GitHub 연동을 설치하면,
  PR 자동 리뷰나 `@claude` 멘션으로 이슈→PR이 가능해진다. 코워크가 연 PR을 클로드 코드가
  리뷰/보완하는 흐름에 적합.
- **C. 로컬 CLI 루프**: 컴퓨터를 켜두고 클로드 코드 CLI에서 `/loop`로 반복. 24시간 무인엔
  A가 더 안정적.

## 활성화 후 협업 방식

`COLLAB.md`의 프로토콜로 전환: 클로드 코드 = 코드(`packages/**`,`apps/**`), 코워크 =
기획·리서치·리뷰·문서(`docs/**` 등). 할 일은 `BACKLOG.md`. 둘 다 브랜치+PR로 작업.

## 참고: 깃허브 없이도 최소 백업은 가능

당장 깃허브 세팅이 어렵다면, 코워크가 저장소를 주기적으로 **이 Claude 프로젝트("공부")에
스냅샷으로 저장**할 수 있다(프로젝트는 세션이 끝나도 유지됨). 완전한 협업은 안 되지만
"작업 유실" 위험은 크게 줄어든다. 원하면 말해주세요.
